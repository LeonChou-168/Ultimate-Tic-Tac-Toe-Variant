# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please avoid opening a public issue with sensitive exploit details.

Instead, report it privately to the repository maintainer if possible, and include:

- A description of the issue
- Reproduction steps or proof of concept
- Affected files, commands, or runtime paths
- Any suggested mitigation if known

## Scope

This project is currently a frontend-focused game implementation with local tooling. Relevant reports may include:

- Dependency-related vulnerabilities
- Unsafe local command behavior
- Exposed secrets or sensitive files accidentally committed
- Security-sensitive UI or networking behavior if/when online features are added

## Response Expectation

The project does not currently guarantee enterprise-style response SLAs, but good-faith reports will be reviewed and prioritized when possible.
